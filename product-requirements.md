# 个人生活助手 — MVP 产品需求文档

> 版本：v0.1（MVP）  
> 更新日期：2026-06-05  
> 定位：个人使用的在线笔记、日记与待办助手，支持多端访问与结构化记录

---

## 1. 核心用户场景

### 场景 A：每日记录与任务管理（最高频）

**用户故事**：我每天打开应用，写下今天做了什么（日记），列出今天要完成的任务（待办），做完就勾选；晚上回顾时，给未完成的待办补一句说明——完成了多少、为什么没做完。

**典型流程**：

1. 早晨：查看今日待办，必要时从计划里拆几条任务过来
2. 白天：随时追加日记片段、勾选已完成待办
3. 晚上：写当日日记总结，对未完成待办填写完成情况说明

**成功标准**：一天内的日记与待办在同一「今日视图」里完成，无需切换多个复杂页面。

---

### 场景 B：知识沉淀（笔记）

**用户故事**：我学到一项技术或踩了一个坑，想快速记下来，打上标签，以后能搜到。

**典型流程**：

1. 新建笔记，填写标题与正文（支持 Markdown）
2. 选择分类（如「前端」「AI」）和标签（如「React」「易错点」）
3. 日后通过关键词或标签检索

**成功标准**：30 秒内完成一条笔记录入；检索能在 3 秒内返回相关结果。

---

### 场景 C：计划与执行（简化版）

**用户故事**：我有一个短期或长期计划（比较模糊、宏大），先记下来；MVP 阶段手动把计划拆成待办，不依赖 AI。

**典型流程**：

1. 创建计划，填写标题、描述、周期（短期 / 长期）、目标日期
2. 手动添加关联待办，或在待办里关联所属计划
3. 在计划详情页查看关联待办的完成进度

**成功标准**：计划与待办有明确关联，能一眼看到进度概览。

---

### 场景 D：复盘（MVP 仅每日）

**用户故事**：一天结束时，我对照今日待办和日记，写一段复盘：今天完成了什么、有什么收获、明天要注意什么。

**典型流程**：

1. 系统根据当日日记、待办完成情况生成复盘草稿（仅聚合展示，不调用 AI）
2. 用户编辑后保存为「每日复盘」
3. 在复盘列表按日期浏览历史

**成功标准**：复盘创建时能自动引用当日数据，减少重复录入。

---

### 场景 E：跨设备访问

**用户故事**：我在公司电脑和手机浏览器上都能登录同一账号，看到相同的数据。

**典型流程**：

1. 邮箱 + 密码注册 / 登录
2. 响应式 Web 页面，手机与桌面均可正常使用
3. JWT 维持登录态，过期后重新登录

**成功标准**：核心 CRUD 在移动端可用，无需单独开发原生 App。

---

### 场景 F：检索与回顾（MVP 基础版）

**用户故事**：我想找上个月关于「Vue」的笔记，或某段时间的所有日记。

**典型流程**：

1. 进入搜索页，输入关键词
2. 按类型（日记 / 待办 / 笔记 / 计划 / 复盘）、标签、时间范围筛选
3. 点击结果跳转详情

**成功标准**：支持关键词 + 标签 + 时间范围组合筛选；暂不要求 AI 总结。

## 2. 第一版必须做的功能

### 2.1 用户认证

| 功能 | 说明 |
|------|------|
| 邮箱注册 | 邮箱格式校验，密码强度基本要求（≥8 位） |
| 邮箱登录 | 邮箱 + 密码 |
| 密码安全 | 服务端 bcrypt（或 argon2）加密存储，不明文 |
| JWT 鉴权 | Access Token + 合理过期时间；受保护 API 需携带 Token |
| 登出 | 客户端清除 Token；可选服务端 Token 黑名单（MVP 可仅客户端） |

### 2.2 日记

| 功能 | 说明 |
|------|------|
| 创建 / 编辑 / 删除 | 富文本或 Markdown 正文 |
| 按日期组织 | 默认「一天一条」，允许多条（带时间戳） |
| 心情 / 标签 | 可选心情枚举 + 自由标签 |
| 今日视图 | 首页或日记页展示当天内容 |

### 2.3 待办

| 功能 | 说明 |
|------|------|
| CRUD | 标题、描述、截止日期、优先级 |
| 完成勾选 | 标记完成 / 取消完成 |
| 完成说明 | 未完成或部分完成时，可填写进度说明（MVP 用文本字段） |
| 关联计划 | 可选关联到某个计划 |
| 今日待办 | 默认展示当天及未完成的历史待办 |

### 2.4 计划

| 功能 | 说明 |
|------|------|
| CRUD | 标题、描述、类型（短期 / 长期）、起止日期 |
| 进度概览 | 关联待办完成数 / 总数 |
| 手动拆待办 | 从计划页创建关联待办（**不含 AI 自动拆分**） |

### 2.5 复盘（仅每日）

| 功能 | 说明 |
|------|------|
| 创建每日复盘 | 绑定日期，正文 Markdown |
| 引用当日数据 | 展示当日待办完成情况、日记摘要（手动编辑区） |
| 历史列表 | 按日期倒序浏览 |

### 2.6 笔记

| 功能 | 说明 |
|------|------|
| CRUD | 标题、Markdown 正文 |
| 分类 | 预设 + 用户自定义分类 |
| 标签 | 多标签 |
| 技术笔记场景 | 支持「心得」「易错点」等通过标签区分 |

### 2.7 长期目标（简化版）

| 功能 | 说明 |
|------|------|
| CRUD | 标题、描述、状态（进行中 / 已完成 / 搁置） |
| 关联计划 | 可选关联一个或多个计划 |
| 列表展示 | 独立页面浏览，MVP 不做复杂进度追踪 |

### 2.8 检索

| 功能 | 说明 |
|------|------|
| 关键词搜索 | 标题 + 正文全文检索 |
| 筛选 | 内容类型、标签、分类、时间范围 |
| 结果展示 | 分类型列表，点击进详情 |

### 2.9 通用能力

| 功能 | 说明 |
|------|------|
| 标签系统 | 全局标签，跨日记 / 笔记 / 待办 / 计划复用 |
| 响应式 UI | 移动端 + 桌面端 Web 适配 |
| 基础视觉 | 现代简洁风格，深色模式可选（MVP 至少一套主题） |

## 3. 暂时不做的功能

以下功能在完整愿景中重要，但 **不纳入 MVP**，避免第一版范围失控：

| 功能 | 原因 | 建议阶段 |
|------|------|----------|
| 微信文件传输助手 / 微信 Bot 录入 | 依赖微信开放平台、合规与运维成本高 | Phase 2 |
| AI 计划自动拆分为待办 | 需接入 LLM、提示词与交互设计 | Phase 2 |
| AI 检索总结 / 智能问答 | 非 MVP 核心闭环 | Phase 2 |
| 周 / 月 / 季 / 年复盘 | 先验证每日复盘习惯，再扩展周期 | Phase 2 |
| 计划周期内专项复盘 | 依赖多周期复盘体系 | Phase 2 |
| 原生 iOS / Android App | 响应式 Web 已覆盖多端 | Phase 3 |
| 多人协作 / 分享 | 个人工具，无社交需求 | 不做或远期 |
| 附件 / 图片上传 | 增加存储与 CDN 复杂度 | Phase 2 |
| 数据导入导出 | MVP 数据量小，手动迁移可接受 | Phase 2 |
| 邮件验证 / 找回密码 | 可先简化注册流程 | Phase 2 |
| OAuth 第三方登录 | 邮箱密码已满足个人使用 | Phase 3 |
| 离线模式 / PWA | 增强体验，非首日必需 | Phase 2 |
| 提醒 / 通知推送 | 依赖消息通道 | Phase 2 |
| 操作审计日志 | 个人单用户场景优先级低 | Phase 3 |

## 4. 数据实体

### 4.1 实体关系概览

```
User
 ├── DiaryEntry      (1:N)
 ├── Todo              (1:N)
 ├── Plan              (1:N)
 ├── Review            (1:N)
 ├── Note              (1:N)
 ├── LongTermGoal      (1:N)
 └── Tag               (1:N, 用户级标签库)

Plan ──< Todo           (1:N, 可选关联)
LongTermGoal ──< Plan   (N:M, 可选关联)
Tag ──< *               (N:M, 多态关联到 Diary/Todo/Note/Plan 等)
```

## 5. 页面结构

### 5.1 信息架构

```
/                          → 重定向至 /today 或 /login
/login                     → 登录
/register                  → 注册

/today                     → 今日主页（核心入口）
/diary                     → 日记列表
/diary/:id                 → 日记详情 / 编辑
/todos                     → 待办列表（含筛选：今日 / 全部 / 已完成）
/todos/:id                 → 待办详情 / 编辑
/plans                     → 计划列表
/plans/:id                 → 计划详情（含关联待办与进度）
/reviews                   → 复盘列表
/reviews/:id               → 复盘详情 / 编辑
/reviews/new?date=         → 新建每日复盘（可预填当日数据）
/notes                     → 笔记列表（按分类 / 标签筛选）
/notes/:id                 → 笔记详情 / 编辑
/goals                     → 长期目标列表
/goals/:id                 → 长期目标详情
/search                    → 全局搜索
/settings                  → 账户设置（改昵称、改密码、主题）
```

### 5.2 布局结构

**桌面端**

```
┌─────────────────────────────────────────────────────────┐
│  Logo    [今日] [日记] [待办] [计划] [复盘] [笔记] [目标]  🔍  👤 │
├──────────┬──────────────────────────────────────────────┤
│          │                                              │
│  侧边栏   │              主内容区                         │
│ （可选    │         （列表 / 编辑器 / 详情）                │
│  标签筛选）│                                              │
│          │                                              │
└──────────┴──────────────────────────────────────────────┘
```

**移动端**

```
┌──────────────────────┐
│  顶栏：标题 + 操作按钮   │
├──────────────────────┤
│                      │
│      主内容区         │
│                      │
├──────────────────────┤
│ 底部 Tab：今日|待办|笔记|更多 │
└──────────────────────┘
```

### 5.3 核心页面说明

| 页面 | 优先级 | 核心内容 |
|------|--------|----------|
| **今日** `/today` | P0 | 当日日记快捷入口、今日待办列表、一键新建、未完成待办补说明 |
| **待办** `/todos` | P0 | 列表 + 筛选 + 快速勾选 |
| **日记** `/diary` | P0 | 日历或时间线列表，点击进入编辑 |
| **笔记** `/notes` | P0 | 分类侧栏 + 笔记卡片列表 |
| **搜索** `/search` | P0 | 搜索框 + 筛选器 + 分类型结果 |
| **登录 / 注册** | P0 | 表单 + 错误提示 |
| **计划** `/plans` | P1 | 计划卡片 + 进度条 |
| **复盘** `/reviews` | P1 | 日期列表 + 新建复盘 |
| **长期目标** `/goals` | P1 | 目标卡片 + 状态 |
| **设置** `/settings` | P1 | 账户与主题 |

### 5.4 UI 风格方向（MVP）

- **基调**：深色为主或支持深浅切换，偏「个人工作台」而非传统博客
- **组件**：卡片式布局、微动效（完成待办时的勾选动画）、清晰 typography
- **编辑器**：Markdown 预览分栏或所见即所得二选一（MVP 可先做 Markdown + 预览）
- **数据可视化**：计划进度条、今日待办完成率（简单数字 / 环形图即可）

## 6. 后续扩展方向

### Phase 2 — 输入与智能

1. **微信接入**
   - 微信 Bot / 企业微信 / 公众号消息回调
   - 支持发送文字 → 自动创建日记或待办
   - 文件传输助手场景：解析文本消息入库

2. **AI 能力**
   - 计划 → 待办自动拆分（可编辑确认后再入库）
   - 按时间段 / 标签的 content 总结
   - 复盘草稿 AI 生成（基于当日日记 + 待办）

3. **多周期复盘**
   - 周 / 月 / 季 / 年复盘模板
   - 计划周期结束后的专项复盘报告

4. **体验增强**
   - 图片 / 附件上传
   - PWA 离线缓存
   - 邮件验证与密码找回
   - 数据导出（Markdown / JSON）

### Phase 3 — 平台化

1. 原生移动端（或 uni-app / React Native）
2. 桌面端 Electron 壳
3. 提醒推送（浏览器 Notification / 移动端推送）
4. 数据备份与多设备同步优化
5. 插件化：自定义字段、模板市场

### Phase 4 — 可选远期

1. 知识图谱：笔记 / 日记之间的关联推荐
2. 公开博客模式：选择性将笔记发布为对外页面
3. API 开放：供其他自动化工具（如 Shortcuts、n8n）调用

## 附录：MVP 验收标准（摘要）

- [ ] 用户可注册、登录、登出，密码加密存储，API 受 JWT 保护
- [ ] 可在今日页完成日记写入与待办勾选
- [ ] 待办支持填写完成 / 未完成说明
- [ ] 笔记支持分类、标签，并可被搜索命中
- [ ] 计划可创建并手动关联待办，展示进度
- [ ] 可创建每日复盘并查看历史
- [ ] 长期目标可 CRUD 并关联计划
- [ ] 搜索支持关键词 + 类型 + 标签 + 时间筛选
- [ ] 手机与桌面浏览器均可完成上述操作
- [ ] UI 具备统一的现代视觉风格，非纯功能堆砌
